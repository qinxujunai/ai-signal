Stainless CEO Alex Rattray详解MCP服务器设计原则：保持工具集精简，为每个工具赋予精确名称和描述 · Stainless CEO Alex Rattray on MCP Server Design: Keep Tool Sets Lean, Give Each Tool a Precise Name and Description
在Anthropic以3亿美元收购Stainless之前，CEO Alex Rattray在播客中详细阐述了MCP（模型上下文协议）服务器的设计哲学。核心原则包括：保持工具集精简、为每个工具赋予精确名称和描述、最小化模型需要处理的输入输出。对于大型API，Stainless采用“动态模式”，只给模型三个工具：列出端点、选择并学习一个端点、执行它。Rattray认为MCP的未来在于代码执行——给模型一个简单的代码执行工具和一个文档搜索工具，让AI自己写代码调用API的SDK。
Before Anthropic acquired Stainless for $300M, CEO Alex Rattray detailed MCP server design philosophy in a podcast. Key principles: keep tool sets lean, give each tool a precise name and description, minimize model input/output. For large APIs, Stainless uses 'dynamic mode' giving models only three tools: list endpoints, pick and learn one, execute it. Rattray believes MCP's future is code execution—give models a simple code execution tool and a doc search tool, letting AI write code against API SDKs.
